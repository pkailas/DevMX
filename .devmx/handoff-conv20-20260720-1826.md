# Project handoff

## What this project is
- **VerbellaCMG Case Management** ‚Äî a case‚Äëmanagement web application.
- **Backend:** ASP.NET Core Web API (.NET 10), EF Core, SQL Server (`VerbellaCMG_CaseManagement_PRD` on `WIN-SQL002\MSSQLSERVER01`).
- **Frontend:** React SPA served from `wwwroot` inside the API process.
- **Auth:** ASP.NET Core Identity (JWT) + optional Windows Negotiate.
- **Deployment targets:**
  - Staging publish folder: `C:\deploy\verbella-cm`
  - Production folder: `C:\Program Files\Verbella CMG, LLC\VerbellaCMG Case Management`
  - IIS site: `http://support.ionline.com:8081`, app pool `VCMG_CaseManagement`

## Key decisions made
- **Email provider:** Gmail SMTP (`smtp.gmail.com:587`) is in use; IMAP polling is enabled but Microsoft Graph is not configured.
- **Credential storage:** SMTP/IMAP passwords must be managed via the app‚Äôs admin settings UI (`/api/settings/email`) ‚Äì they are currently stored in the `EmailSettings` database row (plaintext) and also read from `appsettings.json` on first seed.
- **Versioning:** Automated version bump (3rd ordinal) was implemented in an earlier session (no changes now, but it remains the rule).
- **Encryption approach (under development):** Windows DPAPI (`ProtectedData`) using `DataProtectionScope.CurrentUser`, with a `!ENC!` prefix marking encrypted values in `appsettings.json`. This is intended to protect passwords in the JSON file and to sync DB‚ÜíJSON on every settings save.
