## Key decisions made

1. **Broken migration `FixSlaSettingsIdentity`** – The `.Designer.cs` referenced a `Customer.Cases` navigation that no longer exists in the model. Since the migration’s SQL had already been applied to the database, we chose **not** to restore the source file; instead we **manually inserted** a record into `__EFMigrationsHistory` so EF Core believes it is applied.
2. **`PortalSettings.Id` identity conflict** – The database column was defined with `IDENTITY(1,1)` but the seed code explicitly inserts `Id = 1`. We added `[Key]` and `[DatabaseGenerated(DatabaseGeneratedOption.None)]` attributes to the entity, and **manually altered the database** to drop the identity property (dropping and re‑adding the column as a plain `int` with a primary‑key constraint).
3. **Synchronisation after manual DB changes** – We generated a new migration (`FixPortalSettingsIdentity`), which turned out to be empty (model already matches DB), then deleted it, leaving the snapshot and the database perfectly aligned.
4. **Zero‑warning policy** – All builds must succeed with 0 warnings; the final build/publish was verified for this.
